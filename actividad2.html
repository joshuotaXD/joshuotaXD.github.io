Preguntas Clase 2

¿Qué pasaría si una aplicación popular tuviera algoritmos ineficientes?

El impacto sería crítico tanto para el usuario como para la empresa. Una aplicación con millones de usuarios experimentaría tiempos de carga excesivos, drenaje rápido de la batería del dispositivo y bloqueos constantes (lag). Para la empresa, esto se traduce en costos operativos altísimos, ya que necesitarían más potencia de servidores para compensar la ineficiencia, y finalmente, la pérdida de usuarios frente a la competencia.

Influencia del tamaño de los datos en el diseño

El tamaño de los datos es el factor que determina la arquitectura del software. No se puede diseñar igual una app que maneja 100 usuarios a una que maneja millones:

  • Selección de Estructuras: A mayor volumen, se requieren estructuras de datos más complejas (como árboles o tablas hash) para búsquedas rápidas.
  • Escalabilidad: El software debe diseñarse para ser "escalable", permitiendo que el sistema crezca sin necesidad de reescribir todo el código.
  • Uso de Memoria: Con datos masivos, el diseño debe priorizar algoritmos que optimicen el uso de la RAM y reduzcan el tráfico de red.
  • Experiencia de Usuario: Influye en la decisión de usar técnicas como la "paginación" o "carga diferida" (lazy loading) para no saturar la interfaz.
