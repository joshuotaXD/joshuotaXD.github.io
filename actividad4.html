Descripción de la Actividad

Análisis de la eficiencia de los algoritmos mediante la medición del tiempo de ejecución y el uso de recursos conforme aumenta el volumen de datos de entrada dentro de un sistema

Notación Big O

La notación Big O es un lenguaje matemático que se utiliza para describir la complejidad de un algoritmo permitiendo comparar qué tan eficiente es un código frente a otro al procesar grandes cantidades de información

Tiempo constante O(1) ocurre cuando el tiempo de ejecución no cambia sin importar qué tan grande sea el conjunto de datos como cuando accedes directamente a un elemento en un arreglo

Crecimiento lineal O(n) sucede cuando el tiempo de ejecución aumenta de manera proporcional al tamaño de los datos como cuando necesitas recorrer toda una lista para encontrar un valor específico

Crecimiento cuadrático O(n²) se presenta cuando el tiempo de ejecución aumenta de forma drástica al crecer los datos siendo típico de algoritmos que tienen ciclos anidados como algunos métodos de ordenamiento simples

Crecimiento logarítmico O(log n) es un nivel de eficiencia muy alto donde el tiempo aumenta de forma muy lenta respecto al crecimiento de los datos tal como sucede en la búsqueda binaria donde el espacio de búsqueda se divide a la mitad en cada paso
